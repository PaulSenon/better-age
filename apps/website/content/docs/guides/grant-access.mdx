---
title: Grant access
description: Add or refresh recipients explicitly on the payload.
---

Grant is the operation that adds or refreshes one recipient in a payload.

## 1. Save the identity locally

If you received an identity string, save it first:

```sh
bage add-identity 'better-age://identity/v1/<base64url-json>'
```

That creates a known identity in local home state. It does not mutate any payload yet.

## 2. Grant that identity on the payload

```sh
bage grant .env.prod.enc teammate#0123abcd
```

That writes or refreshes the recipient entry on the payload itself.

## What grant means in the model

- an identity exists independently
- a known identity is a local address-book entry
- a recipient is that identity granted on one payload

If you only save a known identity locally, access has not been granted yet.
