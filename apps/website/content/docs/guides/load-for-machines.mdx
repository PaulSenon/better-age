---
title: Load for machines
description: Keep machine loading explicit and separate from the human secure-viewer path.
---

better-age keeps human and machine plaintext paths distinct on purpose.

## Human path

```sh
bage view .env.prod.enc
```

`view` opens the secure in-process viewer. It is for humans, not for piping plaintext into another process.

## Machine path

```sh
bage load --protocol-version=1 .env.prod.enc
```

`load` is the versioned load protocol. It prints raw `.env` text to `stdout` for another process to consume.

## Why the split matters

- the happy path avoids plaintext leakage as fallback convenience
- machine integrations can depend on one explicit contract
- human reading does not silently degrade into `stdout`

If you want the concrete integration surface built on top of this contract, read [Varlock](/docs/integrations/varlock).
