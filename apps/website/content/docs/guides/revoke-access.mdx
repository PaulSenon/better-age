---
title: Revoke access
description: Remove recipients explicitly instead of hiding access changes inside unrelated commands.
---

Revoke is the operation that removes one recipient from future rewritten versions of a payload.

## Revoke a recipient

```sh
bage revoke .env.prod.enc teammate#0123abcd
```

This is explicit on purpose. better-age avoids hiding access mutation inside unrelated commands or background automation.

## What revoke does not mean

- it does not delete the identity from your local address book
- it does not turn better-age into a hidden sync product
- it does not change the product model: access remains payload-local and caller-owned

If you only want to clean up a saved identity locally, use `forget-identity` instead of `revoke`.
