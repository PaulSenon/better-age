---
title: First-time setup
description: Set up your local identity once, then reuse it across payloads.
---

Your local setup gives better-age its stable identity anchor.

## Run setup

```sh
npx @better-age/cli setup
```

That creates local home state with:

- a stable `ownerId`
- a current keypair
- a passphrase-protected private key
- a display name

## Export your identity string

```sh
bage me
```

The identity string is the versioned one-line export of your identity used by `me` and `add-identity`.

## Save teammates locally

```sh
bage add-identity 'better-age://identity/v1/<base64url-json>'
bage identities
```

Known identities are local address-book entries. Saving them does not mutate any payload by itself.

## Mental model checkpoint

After first-time setup you have:

- one stable local identity
- one shareable identity string
- zero payload access changes so far

Payload access begins later, when someone runs `grant` on an encrypted payload.
