---
title: Create and edit
description: Create one visible encrypted payload file, then edit it locally without hiding the file from the caller.
---

better-age is built around one obvious encrypted payload file.

## Create a payload

```sh
bage create .env.prod.enc
```

## Edit it locally

```sh
bage edit .env.prod.enc
```

The tool keeps the encrypted file visible and caller-owned. It does not turn the payload into a hidden remote registry.

## Inspect metadata without plaintext

```sh
bage inspect .env.prod.enc
```

Use `inspect` when you need non-secret payload metadata and env key names without opening plaintext content.

## Why this guide exists

The product stance is that payload truth lives in the payload itself:

- recipients
- payload metadata
- encrypted env text

That is why create, edit, inspect, grant, revoke, and update stay legible operations instead of disappearing behind a bigger secret-management system.
