---
title: Runtime contract
description: Launcher defaults, process model, and failure semantics for the varlock adapter.
---

## Defaults

- default launcher is `bage`
- protocol version is `1`

## Process model

The plugin spawns:

```txt
bage load --protocol-version=1 <path>
```

If the configured launcher contains spaces, it is treated as a shell launcher prefix.

## Output contract

- child `stdout` becomes env text for varlock
- child `stderr` stays visible in the invoking shell
- non-zero exit becomes plugin failure

## Failure semantics

Typical runtime failures include:

- CLI command failed to start
- plugin used before initialization
- `bage load` exited non-zero
- multiple different init configs attempted in one process
