## Purpose

`update` is the explicit maintenance command.

It exists so schema migration and stale self-recipient refresh stay visible operations instead of hidden background mutation.

## Example

```sh
bage update .env.prod.enc
```

## Behavior

- may resolve the path in guided mode
- prompts for passphrase
- reports `payload already up to date` when nothing changed
- reports the update reasons when a rewrite happened

## Typical reasons

- self key is stale
- payload schema is outdated

Use `update` before commands like `grant` or `revoke` when the CLI tells you the payload needs maintenance first.
