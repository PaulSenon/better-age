title: Installation

---

The shortest path is a one-off invocation:

```sh
npx @better-age/cli setup
```

If you already have the CLI on your path, the same action is:

```sh
bage setup
```

That creates your local identity state and asks for the passphrase that protects your private key material.

## What setup gives you

- one stable `ownerId`
- one current keypair
- one passphrase-protected private key
- one display name
- key mode `pq-hybrid`

## Command shape

Many examples in the docs assume `bage` is already invokable in your shell:

```sh
bage setup
bage me
```

If you are not using a global install or wrapper yet, keep using the package entrypoint style:

```sh
npx @better-age/cli setup
npx @better-age/cli me
```

## What to do next

After setup:

1. export your identity string with `bage me`
2. create a payload with `bage create .env.enc`
3. read the full [Quickstart](/docs/getting-started/quickstart)

If you want the rationale behind the workflow before running commands, go back to [Vision](/docs/getting-started/vision).
