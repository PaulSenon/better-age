**better-age** is a local-first secret provider for Varlock.

It is for small teams already passing `.env` files around, but ready to stop keeping shared plaintext secrets on disk and in chat history. You still work with one visible env-like file. It is just encrypted, recipient-based, and backed by age.

<Callout type="info" title="What is age?">
  [age](https://age-encryption.org/) is a small modern file-encryption tool and format. **better-age** is not an official age project and is not affiliated with the age maintainers. It uses the age ecosystem as the crypto foundation and adds the `.env` workflow around it. Read [Age 101](/docs/guides/age-101) for the basic mental model.
</Callout>

The shortest path:

```sh
npm install -g @better-age/cli
bage setup --name Alice
bage create .env.enc
bage edit .env.enc
```

Then wire Varlock so your app runs from the encrypted payload instead of a plaintext `.env`.

## Product stance

- `age` is the crypto primitive.
- Typage is the TypeScript age implementation **better-age** relies on.
- **better-age** is the UX and workflow layer.
- Varlock is the recommended runtime path.
- Scope stays narrow: encrypted env payloads, not generic secret management.

<Callout type="info">
  **better-age** is not trying to replace a full cloud secret manager. It is a better default before your team needs that level of system.
</Callout>
