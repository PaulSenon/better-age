better-age is a local-first workflow for encrypted `.env` payloads.

It is built around one narrow job:

- create one visible encrypted payload file
- edit and read it locally
- grant or revoke recipients explicitly
- keep machine loading explicit

## Start here

If you just want first success, start with setup:

```sh
npx @better-age/cli setup
```

Then move through:

- [Vision](/docs/getting-started/vision)
- [Installation](/docs/getting-started/installation)
- [Quickstart](/docs/getting-started/quickstart)

That path is the intended onboarding order. Read the why first, install second, then run the smallest end to end flow.

The core happy path stays small:

```sh
bage create .env.prod.enc
bage edit .env.prod.enc
bage grant .env.prod.enc teammate#0123abcd
bage view .env.prod.enc
bage load --protocol-version=1 .env.prod.enc
```

## Search and AI surfaces

- search is built into the docs shell for fast page lookup
- `/llms.txt` exposes the docs index for AI tooling
- `/llms-full.txt` exposes a fuller concatenated text surface
- each docs page also has a markdown-friendly export path

These surfaces are intentionally part of the site contract, not incidental extras.

## Product stance

- `age` is the primitive
- `better-age` is the UX layer
- scope stays narrow on env files, not generic secret management
- `view` is the human plaintext path
- `load` is the machine plaintext path
